# Data Model: Bounded Agent Runtime Loop

All persisted artifacts include `schema_version`. IDs must be stable within a
run and safe for local JSON/JSONL storage.

## Enums

### RuntimeMode

- `phase1_local`
- `replay`
- `debug`

Phase 1 production-like runs use `phase1_local`. `debug` enables extra local
diagnostics such as full prompt logging. `replay` reconstructs from saved
artifacts without model calls.

### ToolName

- `validate_scenario_input`
- `inspect_missing_facts`
- `check_rule_coverage`
- `load_reviewed_rule_pack`
- `query_reviewed_regulation_context`
- `generate_rule_draft_candidate`
- `record_rule_draft_candidate`
- `validate_judgment_draft`
- `record_agent_trace`
- `create_human_review_task`

### ForbiddenAction

- `parse_pdf`
- `parse_docx`
- `run_001_structuring_pipeline`
- `activate_rule_pack`
- `promote_rule_draft`
- `write_active_rule`
- `write_reviewed_regulation_context`
- `produce_final_compliance_conclusion`
- `use_draft_rule_as_formal_basis`
- `call_web_search`

### ValidationStatus

- `pending`
- `accepted`
- `rejected`
- `skipped`

### RuleCoverageStatus

- `matched`
- `no_matching_rule`

MVP fully supports `no_matching_rule`. `matched` remains schema-supported for
future extension.

### ReviewedContextStatus

- `not_checked`
- `available`
- `insufficient`
- `not_available`

### JudgmentStatus

- `needs_review`
- `insufficient_rule_coverage`
- `blocked`
- `error`

When `rule_coverage_status = no_matching_rule`, final-looking statuses such as
`approved`, `final`, `pass`, or `compliant` are invalid.

### ReviewStatus

- `needs_review`
- `approved`
- `rejected`
- `superseded`

`RuleDraftCandidate` generated by Spec002 always uses `needs_review`.

### StopReason

- `completed`
- `no_matching_rule`
- `human_review_required`
- `max_rounds_exceeded`
- `max_tool_calls_exceeded`
- `max_token_budget_exceeded`
- `validation_failed`
- `model_error`
- `tool_error`
- `forbidden_action_rejected`
- `cancelled`

## Entities

### UserInput

Raw user-submitted input before scenario normalization.

| Field | Type | Required | Notes |
|-------|------|----------|-------|
| `input_id` | string | yes | Stable input ID |
| `raw_text` | string | yes | Sensitive by default; not saved raw unless explicitly allowed |
| `structured_fields` | object | yes | User-provided structured facts, may be empty |
| `submitted_at` | datetime | yes | Submission timestamp |
| `user_context` | object | yes | Non-secret context such as source channel or user role |
| `redaction_status` | string | yes | `redacted`, `not_redacted`, or `not_saved` |

Validation rules:
- `raw_text` must not be persisted in local run artifacts by default.
- `user_input.redacted.json` may store redacted text or summaries.

### ScenarioInput

Structured runtime input available before AgentLoop starts.

| Field | Type | Required | Notes |
|-------|------|----------|-------|
| `scenario_id` | string | yes | Stable scenario ID |
| `normalized_question` | string | yes | Normalized compliance question |
| `business_subject` | string or null | no | Subject of the scenario |
| `action_type` | string or null | no | Action being evaluated |
| `amount` | object or null | no | Amount value/currency/notes when known |
| `date_fields` | object | yes | Relevant date facts |
| `as_of_date` | date or null | no | Temporal interpretation date |
| `known_facts` | object | yes | Facts available to the Agent |
| `missing_facts` | list[string] | yes | Missing facts surfaced before or during runtime |
| `validation_status` | string | yes | Scenario validation result |

Validation rules:
- AgentLoop starts only after a `ScenarioInput` exists.
- Missing or uncertain facts remain explicit; they are not fabricated.

### AgentRuntimeState

Mutable state used by the Agent to decide and by runtime to enforce limits.

| Field | Type | Required | Notes |
|-------|------|----------|-------|
| `run_id` | string | yes | Shared run ID |
| `scenario_id` | string | yes | Links to `ScenarioInput` |
| `current_round` | integer | yes | Current round index |
| `remaining_rounds` | integer | yes | Decremented per round |
| `remaining_tool_calls` | integer | yes | Decremented per accepted tool call |
| `remaining_token_budget` | integer or null | no | Configurable token budget |
| `allowed_tools` | list[ToolName] | yes | Phase 1 allowed tools |
| `completed_actions` | list[string] | yes | Accepted action IDs |
| `observations` | list[object] | yes | Tool and validation observations |
| `rule_coverage_status` | RuleCoverageStatus or null | no | Latest coverage status |
| `reviewed_context_status` | ReviewedContextStatus | yes | Latest reviewed context status |
| `generated_draft_rule_ids` | list[string] | yes | Exploratory draft candidate IDs |
| `judgment_draft_id` | string or null | no | Linked output draft |
| `human_review_required` | boolean | yes | Runtime review gate |
| `stop_reason` | StopReason or null | no | Set when stopped |
| `validation_errors` | list[string] | yes | Current validation errors |
| `warnings` | list[string] | yes | Runtime warnings |

State rules:
- Runtime updates state after every accepted tool call, rejection, validation
  failure, budget event, warning, or stop.
- The Agent may observe state but cannot mutate it directly.

### NextActionPlan

Agent-proposed plan for the next action.

| Field | Type | Required | Notes |
|-------|------|----------|-------|
| `action_id` | string | yes | Stable action ID |
| `run_id` | string | yes | Linked run ID |
| `round_index` | integer | yes | Round that produced the plan |
| `selected_tool` | string | yes | Must be allowed before execution |
| `purpose` | string | yes | What the Agent is trying to accomplish |
| `expected_input` | object | yes | Proposed tool input summary/schema-shaped payload |
| `expected_output` | object | yes | Expected output summary/schema |
| `why_this_action` | string | yes | Agent rationale for this step |
| `current_assumption` | string | yes | Assumption being tested or used |
| `risk_notes` | list[string] | yes | Known risks or uncertainty |
| `requires_validation` | boolean | yes | Always true for tool execution |
| `stop_if_success` | string or null | no | Agent-proposed stop condition |
| `stop_if_failure` | string or null | no | Agent-proposed failure handling |
| `validation_status` | ValidationStatus | yes | Set by StepValidator |
| `rejection_reason` | string or null | no | Set when rejected |
| `created_at` | datetime | yes | Plan timestamp |

Validation rules:
- The Agent cannot execute tools directly.
- `selected_tool` must be in `allowed_tools`.
- Forbidden actions are rejected even if expressed as a tool-like string.

### RuleCoverageReport

Result of active reviewed rule coverage check.

| Field | Type | Required | Notes |
|-------|------|----------|-------|
| `coverage_status` | RuleCoverageStatus | yes | `matched` or `no_matching_rule` |
| `matched_rule_ids` | list[string] | yes | Empty for MVP no-matching-rule path |
| `checked_rule_pack_ids` | list[string] | yes | Reviewed packs checked |
| `reason` | string | yes | Human-readable explanation |
| `confidence` | number or null | no | Bounded score when available |
| `created_at` | datetime | yes | Report timestamp |

Validation rules:
- MVP must fully support `no_matching_rule`.
- Draft rule IDs cannot appear in `matched_rule_ids`.

Exploratory draft rule material for human review.

| Field | Type | Required | Notes |
|-------|------|----------|-------|
| `draft_rule_id` | string | yes | Stable draft ID |
| `source_run_id` | string | yes | Run that generated it |
| `scenario_id` | string | yes | Source scenario |
| `trigger_reason` | string | yes | `no_matching_rule` for MVP |
| `draft_title` | string | yes | Draft title |
| `draft_condition` | string | yes | Possible condition |
| `draft_action` | string | yes | Possible action |
| `draft_obligation` | string or null | no | Possible obligation |
| `draft_exception` | string or null | no | Possible exception |
| `source_evidence` | list[object] | yes | Reviewed context evidence only |
| `generated_rationale` | string | yes | Why candidate was generated |
| `confidence` | number or null | no | Bounded score when available |
| `review_status` | ReviewStatus | yes | Always `needs_review` when generated |
| `created_at` | datetime | yes | Creation timestamp |
| `human_review_required` | boolean | yes | Always true |

Validation rules:
- Must be separate from `RuleItem`.
- Must not be active, promoted, or compatible with active `RulePack` without a
  later explicit promotion workflow.
- Must not be used as formal judgment basis.

### JudgmentDraft

Conservative runtime output.

| Field | Type | Required | Notes |
|-------|------|----------|-------|
| `judgment_id` | string | yes | Stable judgment draft ID |
| `run_id` | string | yes | Linked run |
| `scenario_id` | string | yes | Linked scenario |
| `judgment_status` | JudgmentStatus | yes | Conservative status |
| `rule_coverage_status` | RuleCoverageStatus | yes | Coverage status |
| `summary` | string | yes | Draft summary |
| `reasoning_summary` | string | yes | Sanitized reasoning summary |
| `formal_basis_rule_ids` | list[string] | yes | Reviewed rule IDs only |
| `exploratory_rule_draft_ids` | list[string] | yes | Draft candidates for review |
| `citations` | list[object] | yes | Reviewed context citations where available |
| `warnings` | list[string] | yes | Review and coverage warnings |
| `validation_report` | object | yes | Output validation result |
| `human_review_required` | boolean | yes | True when no reviewed rule matches |
| `created_at` | datetime | yes | Creation timestamp |

Validation rules:
- `formal_basis_rule_ids` must not contain draft rule IDs.
- `exploratory_rule_draft_ids` may contain `RuleDraftCandidate` IDs.
- If `rule_coverage_status = no_matching_rule`, `judgment_status` must not be
  approved, final, pass, compliant, or equivalent.
- If no reviewed rule matches, `human_review_required` must be true.

## State Transitions

1. `UserInput` is normalized into `ScenarioInput`.
2. `AgentRun` and initial `AgentRuntimeState` are created.
3. In each round, the Agent produces `NextActionPlan`.
4. StepValidator sets `validation_status` to `accepted` or `rejected`.
5. Accepted tool calls update traces, observations, budgets, and runtime state.
6. Rejected actions update traces, warnings/errors, and runtime state.
7. If no reviewed rule matches, runtime state records `no_matching_rule`.
8. Exploratory `RuleDraftCandidate` may be recorded only as `needs_review`.
9. `JudgmentDraft` is validated conservatively.
10. `HumanReviewTask` is created when review is required.
11. `AgentRun.stop_reason` and final `runtime_state.json` are saved.

## Local Run Store

MVP uses `.arbiter/runs/<run_id>/`.

```text
.arbiter/runs/<run_id>/
|-- run.json
|-- user_input.redacted.json
|-- scenario_input.json
|-- runtime_state.json
|-- next_action_plans.jsonl
|-- coverage_report.json
|-- steps.jsonl
|-- tool_calls.jsonl
|-- model_calls.jsonl
|-- token_usage.jsonl
|-- rule_draft_candidates.jsonl
|-- judgment_draft.json
|-- validation_report.json
\-- human_review_tasks.jsonl
```

Storage rules:
- File structure is stable.
- Every artifact includes `schema_version`.
- Sensitive raw input is not saved by default.
- Redacted snapshots may be saved.
- Audit and debug share the same `run_id`.
- Replay must be possible from saved artifacts without model calls.
